# GITHUB WORKFLOW & DOCUMENTATION

## Branching Strategy (Git Flow):

- **Main**: Production-ready code (tagged versions)
- **Develop**: Integration branch (deployment target)
- **feature**: New features (feature/feature-name)
- **Bugfix**: Bug fixes (bugfix/bug-name)
- **Release**: Release prep (release/v1.0.0)
- **hotfix**: Production fixes (hotfix/patch-name)

## PR Process:

1. **Create feature branch from develop**
   - Start from the latest development branch.
2. **Make commits and push**
   - Follow meaningful commit message conventions.
3. **Open PR with template**
   - Link any related issues and describe changes.
4. **Code review by team member**
   - Address feedback and ensure tests pass.
5. **Merge when approved**
   - Use squash and merge or rebase merging.
6. **Delete branch after merge**
   - Keep the repository clean.
